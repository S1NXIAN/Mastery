# Mastery

Curated, self-paced learning paths designed to build programming language mastery and clean software engineering practices.

## Why books?

Honestly, YouTube tutorials and interactive coding websites just don't cut it. 

Videos are great if you want a quick 10-minute overview of a concept, but they tend to skip the hard stuff to keep you watching. Interactive coding sites are fun for a bit, but they hold your hand way too much with custom sandboxes. They don't actually teach you how to write real code, configure your compiler, or debug an actual environment on your own machine.

I prefer books because they don't skip details to stay engaging. They force me to actually set things up, think about what I'm writing, and figure things out when they break. It's slower, but it's the only way I've found to actually get good at this.
