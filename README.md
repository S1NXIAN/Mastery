# Mastery

Programming curricula built on a single premise: you cannot optimize what you do not understand.
Each roadmap starts at the language and ends at the machine — covering systems programming, memory models, tooling discipline, and architectural thinking that only comes from writing code that breaks until you fix it.

## How this works

Each curriculum specifies a sequence of books, a mandatory toolchain with compiler flags, weekly projects of increasing difficulty, and gate checks at each phase boundary.
Gate checks exist because reading without building produces no useful skill.
If you cannot pass the gate, repeating the work until you can is the point.
